# Policies: Remote Work Security for Medical Staff 🏠🔐

This Standard Operating Procedure (SOP) defines the mandatory security controls for healthcare personnel accessing medical systems, tele-health platforms, or Electronic Health Records (EHR) from outside the clinical facility.

## 1. Device Requirements
*   **Authorized Devices Only:** Medical data must only be accessed using facility-issued or strictly vetted devices. The use of shared family computers is strictly prohibited.
*   **Endpoint Protection:** All remote devices must have active, updated enterprise-grade antivirus and endpoint detection (EDR) software installed.
*   **Disk Encryption:** The hard drives of all laptops used for remote medical work must be fully encrypted (e.g., BitLocker, FileVault).

## 2. Secure Connections
*   **VPN Requirement:** All remote access to the clinic's internal network or EHR systems must be routed through a secure, encrypted Virtual Private Network (VPN).
*   **Home Wi-Fi Standards:** Home networks must be secured with at least WPA2/WPA3 encryption and a strong, non-default router password.

## 3. Data Handling & Storage
*   **No Local Storage:** Staff must never download, save, or cache patient records, images, or financial data on the local hard drive of a remote device.
*   **Secure Disposal:** If physical notes containing Protected Health Information (PHI) are taken during a remote session, they must be securely shredded, not thrown in standard household trash.

# Políticas: Seguridad en el Trabajo Remoto para Personal Médico 🏠🔐

Este Procedimiento Operativo Estándar (SOP) define los controles de seguridad obligatorios para el personal de salud que accede a sistemas médicos, plataformas de telesalud o Historias Clínicas Electrónicas (EHR) desde fuera de las instalaciones clínicas.

## 1. Requisitos del Dispositivo
*   **Solo Dispositivos Autorizados:** Solo se debe acceder a los datos médicos utilizando dispositivos emitidos por el centro o estrictamente examinados. El uso de computadoras familiares compartidas está estrictamente prohibido.
*   **Protección de Endpoints:** Todos los dispositivos remotos deben tener instalado un software antivirus de nivel empresarial y detección de endpoints (EDR) activo y actualizado.
*   **Encriptación de Disco:** Los discos duros de todas las computadoras portátiles utilizadas para trabajo médico remoto deben estar completamente encriptados (ej. BitLocker, FileVault).

## 2. Conexiones Seguras
*   **Requisito de VPN:** Todo el acceso remoto a la red interna de la clínica o a los sistemas EHR debe enrutarse a través de una Red Privada Virtual (VPN) segura y encriptada.
*   **Estándares de Wi-Fi Doméstico:** Las redes domésticas deben estar aseguradas con al menos encriptación WPA2/WPA3 y una contraseña de router robusta y no predeterminada.

## 3. Manejo y Almacenamiento de Datos
*   **Sin Almacenamiento Local:** El personal nunca debe descargar, guardar o almacenar en caché registros de pacientes, imágenes o datos financieros en el disco duro local de un dispositivo remoto.
*   **Eliminación Segura:** Si se toman notas físicas que contengan Información Protegida de Salud (PHI) durante una sesión remota, deben triturarse de forma segura, no tirarse a la basura doméstica estándar.
